# Screenshot capture guide

This repo is set up to support consistent, portfolio-friendly screenshots in GitHub README files.

## Where to put images
- Store screenshots in `assets/screenshots/`
- Keep filenames stable so README links don't break

## Naming convention (recommended)
- `assets/screenshots/01-hero.png`
- `assets/screenshots/02-api-requests.png`
- `assets/screenshots/03-compliance-endpoints.png`
- `assets/screenshots/04-seed-data.png`

## Recommended shots for this repo
1. **API requests** (Postman / REST Client / curl output)
2. **Compliance endpoints** (audit logs + GST calc)
3. **Seed data summary** (console output from `rails db:seed`)
4. **Routes overview** (snippet or diagram)

## Image format
- Prefer PNG
- Target ~1200px wide
